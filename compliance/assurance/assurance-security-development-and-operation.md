<span data-ttu-id="1f6d6-105">Microsoft 的安全性開發生命週期 (SDL) 是一種安全性保證程式，主要是開發及作業安全的軟體。</span><span class="sxs-lookup"><span data-stu-id="1f6d6-105">Microsoft's Security Development Lifecycle (SDL) is a security assurance process focused on developing and operating secure software.</span></span> <span data-ttu-id="1f6d6-106">SDL 為 Microsoft 的開發人員和工程師提供詳細的、可測量的安全性需求，以降低產品和服務中的漏洞數量和嚴重性。</span><span class="sxs-lookup"><span data-stu-id="1f6d6-106">The SDL provides detailed, measurable security requirements for developers and engineers at Microsoft to reduce the number and severity of vulnerabilities in our products and services.</span></span> <span data-ttu-id="1f6d6-107">所有 Microsoft 軟體發展小組都必須遵循 SDL 的需求，而且我們連續更新 SDL，以反映威脅的變化環境、行業最佳作法，以及法規遵從性的法規標準。</span><span class="sxs-lookup"><span data-stu-id="1f6d6-107">All Microsoft software development teams must follow SDL requirements, and we continuously update the SDL to reflect the changing threat landscape, industry best practices, and regulatory standards for compliance.</span></span>

<span data-ttu-id="1f6d6-119">為了支援我們的開發人員在程式碼開發期間和發行之後實施安全性需求，Microsoft 提供了一套安全的開發工具，以自動檢查原始程式碼中的安全性缺陷和弱點。</span><span class="sxs-lookup"><span data-stu-id="1f6d6-119">To support our developers in implementing security requirements during code development and after release, Microsoft provides a suite of secure development tools to automatically check source code for security flaws and vulnerabilities.</span></span> <span data-ttu-id="1f6d6-120">Microsoft 會定義及發行開發人員使用的核准工具清單，例如編譯器和開發環境，以及內建的安全性檢查，可在 Microsoft 組建管線中自動執行。</span><span class="sxs-lookup"><span data-stu-id="1f6d6-120">Microsoft defines and publishes a list of approved tools for our developers to use, such as compilers and development environments, along with the built-in security checks executed automatically within Microsoft build pipelines.</span></span> <span data-ttu-id="1f6d6-121">我們的開發人員使用最新版的已核准工具，利用新的安全性功能。</span><span class="sxs-lookup"><span data-stu-id="1f6d6-121">Our developers use the latest versions of approved tools to take advantage of new security features.</span></span>

<span data-ttu-id="1f6d6-122">在將程式碼簽入到版本分支之前，SDL 必須個別的檢閱者手動進行程式碼審查。</span><span class="sxs-lookup"><span data-stu-id="1f6d6-122">Before code can be checked into a release branch, the SDL requires manual code review by a separate reviewer.</span></span> <span data-ttu-id="1f6d6-123">程式碼檢閱者會檢查是否有編碼錯誤，並確認程式碼變更符合 SDL 和設計需求、傳遞功能和安全性測試，以及可靠執行。</span><span class="sxs-lookup"><span data-stu-id="1f6d6-123">Code reviewers check for coding errors and verify that code changes meet SDL and design requirements, pass functional and security tests, and perform reliably.</span></span> <span data-ttu-id="1f6d6-124">他們也會檢查相關的檔、驗證和相依性，以確保代碼變更已適當地記錄，而且不會造成非預期的副作用。</span><span class="sxs-lookup"><span data-stu-id="1f6d6-124">They also review associated documentation, configs, and dependencies to ensure code changes are documented appropriately and will not cause unintended side-effects.</span></span> <span data-ttu-id="1f6d6-125">如果檢閱者在程式碼檢查期間發現問題，他們可以要求提交者以建議的變更和其他測試重新提交程式碼。</span><span class="sxs-lookup"><span data-stu-id="1f6d6-125">If a reviewer finds problems during code review, they can ask the submitter to resubmit the code with suggested changes and additional testing.</span></span> <span data-ttu-id="1f6d6-126">程式碼檢閱者可能也會決定完全封鎖不符合需求的程式碼。</span><span class="sxs-lookup"><span data-stu-id="1f6d6-126">Code reviewers may also decide to block check-in entirely for code that does not meet requirements.</span></span> <span data-ttu-id="1f6d6-127">當檢閱者認為程式碼符合要求之後，檢閱者會提供核准，在程式碼可以繼續下一個部署階段之前必須先進行核准。</span><span class="sxs-lookup"><span data-stu-id="1f6d6-127">Once the code has been deemed satisfactory by the reviewer, the reviewer provides approval, which is required before code can proceed to the next deployment phase.</span></span>
