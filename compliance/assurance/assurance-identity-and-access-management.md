## <a name="how-does-microsoft-365-protect-production-systems-from-unauthorized-or-malicious-access"></a><span data-ttu-id="9adb6-104">Microsoft 365 如何防止未授權或惡意存取的實際執行系統？</span><span class="sxs-lookup"><span data-stu-id="9adb6-104">How does Microsoft 365 protect production systems from unauthorized or malicious access?</span></span>

<span data-ttu-id="9adb6-105">Microsoft 365 的設計可讓 Microsoft 的工程師在未存取客戶內容的情況下操作服務。</span><span class="sxs-lookup"><span data-stu-id="9adb6-105">Microsoft 365 is designed to allow Microsoft's engineers to operate the service without accessing customer content.</span></span> <span data-ttu-id="9adb6-106">根據預設，Microsoft 365 工程師會將存取權 (ZSA) 客戶內容，而且不會存取實際執行環境。</span><span class="sxs-lookup"><span data-stu-id="9adb6-106">By default, Microsoft 365 engineers have Zero Standing Access (ZSA) to customer content and no privileged access to the production environment.</span></span> <span data-ttu-id="9adb6-107">Microsoft 365 使用即時 (JIT) ，只需足夠存取 (JEA) 模型，以提供服務小組工程師，以在支援 Microsoft 365 時，具有實際的許可權存取實際執行環境。</span><span class="sxs-lookup"><span data-stu-id="9adb6-107">Microsoft 365 uses a Just-In-Time (JIT), Just-Enough-Access (JEA) model to provide service team engineers with temporary privileged access to production environments when such access is required to support Microsoft 365.</span></span> <span data-ttu-id="9adb6-108">JIT 存取模型會取代傳統的持續性系統管理存取，讓工程師在必要時要求暫時提升為特權角色。</span><span class="sxs-lookup"><span data-stu-id="9adb6-108">The JIT access model replaces traditional, persistent administrative access with a process for engineers to request temporary elevation into privileged roles when required.</span></span>

<span data-ttu-id="9adb6-109">指派給服務小組以支援生產服務的工程師會透過身分識別管理工具 (IDM) ，向服務小群組帳戶要求資格。</span><span class="sxs-lookup"><span data-stu-id="9adb6-109">Engineers assigned to a service team to support production services request eligibility for a service team account through the Identity Management Tool (IDM).</span></span> <span data-ttu-id="9adb6-110">要求要求會觸發一系列的人員檢查，以確保工程師已超過所有雲端篩選需求、已完成必要的訓練，以及在建立帳戶之前接收適當的管理核准。</span><span class="sxs-lookup"><span data-stu-id="9adb6-110">The request for eligibility triggers a series of personnel checks to ensure the engineer has passed all cloud screening requirements, completed necessary training, and received appropriate management approval prior to account creation.</span></span> <span data-ttu-id="9adb6-111">只有在符合所有資格需求之後，才可為要求的環境建立服務小群組帳戶。</span><span class="sxs-lookup"><span data-stu-id="9adb6-111">Only after meeting all eligibility requirements can a service team account be created for the requested environment.</span></span>

<span data-ttu-id="9adb6-112">服務小群組帳戶不會授與任何對客戶內容的系統管理員許可權或存取權。</span><span class="sxs-lookup"><span data-stu-id="9adb6-112">Service team accounts do not grant any standing administrator privileges or access to customer content.</span></span> <span data-ttu-id="9adb6-113">當工程師需要其他存取權來支援其 Microsoft 365 服務時，他們會使用稱為「密碼箱」的 access 管理工具，以暫時提升存取所需的資源。</span><span class="sxs-lookup"><span data-stu-id="9adb6-113">When an engineer requires additional access to support their Microsoft 365 service, they request temporary elevated access to the resources they require using an access management tool called Lockbox.</span></span> <span data-ttu-id="9adb6-114">「密碼箱」會限制完成所指派工作所需的最低許可權、資源和時間的許可權。</span><span class="sxs-lookup"><span data-stu-id="9adb6-114">Lockbox restricts elevated access to the minimum privileges, resources, and time needed to complete the assigned task.</span></span> <span data-ttu-id="9adb6-115">如果授權的檢閱者核准 JIT 存取要求，工程師會授與暫存檔，只具有完成所指派工作所需的許可權。</span><span class="sxs-lookup"><span data-stu-id="9adb6-115">If an authorized reviewer approves the JIT access request, the engineer is granted a temporary account with only the privileges necessary to complete their assigned work.</span></span> <span data-ttu-id="9adb6-116">此暫存帳戶需要多重要素驗證，而且會在已核准的期限到期後自動刪除。</span><span class="sxs-lookup"><span data-stu-id="9adb6-116">This temporary account requires multi-factor authentication and is automatically deleted after the approved period expires.</span></span>

## <a name="how-does-microsoft-365-handle-remote-access-to-production-systems"></a><span data-ttu-id="9adb6-117">Microsoft 365 如何處理生產系統的遠端存取？</span><span class="sxs-lookup"><span data-stu-id="9adb6-117">How does Microsoft 365 handle remote access to production systems?</span></span>

<span data-ttu-id="9adb6-118">Microsoft 365 系統元件是存放在與運作小組分開的資料中心。</span><span class="sxs-lookup"><span data-stu-id="9adb6-118">Microsoft 365 system components are housed in datacenters geographically separated from the operations teams.</span></span> <span data-ttu-id="9adb6-119">資料中心人員不具備 Microsoft 365 系統的邏輯存取權。</span><span class="sxs-lookup"><span data-stu-id="9adb6-119">Datacenter personnel do not have logical access to Microsoft 365 systems.</span></span> <span data-ttu-id="9adb6-120">因此，Microsoft 365 服務小組人員會透過遠端存取來管理環境。</span><span class="sxs-lookup"><span data-stu-id="9adb6-120">As a result, Microsoft 365 service team personnel manage the environment through remote access.</span></span> <span data-ttu-id="9adb6-121">需要 Microsoft 365 的遠端存取權的服務小組人員，只會在核准者核准後授與遠端存取權。</span><span class="sxs-lookup"><span data-stu-id="9adb6-121">Service team personnel who require remote access to support Microsoft 365 are only granted remote access after approval from an authorized manager.</span></span> <span data-ttu-id="9adb6-122">所有遠端存取使用 FIPS 140-2 相容的 TLS，以進行安全的遠端連線。</span><span class="sxs-lookup"><span data-stu-id="9adb6-122">All remote access uses FIPS 140-2 compatible TLS for secure remote connections.</span></span>

<span data-ttu-id="9adb6-123">Microsoft 365 使用安全的系統管理工作站 (SAWs) 以取得服務小組遠端存取，以協助保護 Microsoft 365 環境免受損損。</span><span class="sxs-lookup"><span data-stu-id="9adb6-123">Microsoft 365 uses Secure Admin Workstations (SAWs) for service team remote access to help protect Microsoft 365 environments from compromise.</span></span> <span data-ttu-id="9adb6-124">這些工作站專門設計用來避免實際或無意中的實際執行資料遺失，包括鎖定 USB 埠，並將所見即用的軟體限制為支援環境所需的功能。</span><span class="sxs-lookup"><span data-stu-id="9adb6-124">These workstations are specifically designed to prevent intentional or unintentional loss of production data, including locking down USB ports and limiting the software available on the SAW to what is required for supporting the environment.</span></span> <span data-ttu-id="9adb6-125">SAWs 會密切追蹤並加以監控，以偵測 Microsoft 工程師的客戶資料，並防止其惡意或不慎損損。</span><span class="sxs-lookup"><span data-stu-id="9adb6-125">SAWs are closely tracked and monitored to detect and prevent malicious or inadvertent compromise of customer data by Microsoft engineers.</span></span>

## <a name="how-does-customer-lockbox-add-additional-protection-for-customer-content"></a><span data-ttu-id="9adb6-126">客戶的密碼箱如何為客戶內容新增額外的保護？</span><span class="sxs-lookup"><span data-stu-id="9adb6-126">How does Customer Lockbox add additional protection for customer content?</span></span>
