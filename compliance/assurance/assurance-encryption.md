## <a name="what-role-does-encryption-play-in-protecting-customer-content"></a><span data-ttu-id="6fe2d-104">加密在保護客戶內容時會扮演什麼角色？</span><span class="sxs-lookup"><span data-stu-id="6fe2d-104">What role does encryption play in protecting customer content?</span></span>

<span data-ttu-id="6fe2d-105">大部分的 Microsoft business 雲端服務都是多租戶服務，也就是說，客戶內容可能會與其他客戶儲存在相同的實體硬體上。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-105">Most Microsoft business cloud services are multitenant, meaning that customer content may be stored on the same physical hardware as that of other customers.</span></span> <span data-ttu-id="6fe2d-106">為了保護客戶內容的機密性，Microsoft 365 會以所有最強和最安全的加密通訊協定來加密靜態和傳輸中的所有資料。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-106">To protect the confidentiality of customer content, Microsoft 365 encrypts all data at rest and in transit with some of the strongest and most secure encryption protocols available.</span></span>

<span data-ttu-id="6fe2d-107">加密不是強存取控制的替代。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-107">Encryption is not a substitute for strong access controls.</span></span> <span data-ttu-id="6fe2d-108">Microsoft 365 的存取控制原則為零，將存取權 (ZSA) 會保護客戶內容，避免 Microsoft 員工未經授權的存取。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-108">Microsoft 365's access control policy of Zero Standing Access (ZSA) protects customer content from unauthorized access by Microsoft employees.</span></span> <span data-ttu-id="6fe2d-109">加密會在任何地點存取客戶內容的機密性，以及防止內容在 Microsoft 365 系統之間或 Microsoft 365 和客戶之間進行讀取時，加以補充，以進行存取控制。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-109">Encryption complements access control by protecting the confidentiality of customer content wherever it is stored and by preventing content from being read while in transit between Microsoft 365 systems or between Microsoft 365 and the customer.</span></span>

<span data-ttu-id="6fe2d-111">Microsoft 365 中的所有客戶內容都受到一或多個加密形式的保護。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-111">All customer content in Microsoft 365 is protected by one or more forms of encryption.</span></span> <span data-ttu-id="6fe2d-112">Microsoft 伺服器使用 BitLocker 來加密包含磁片磁碟機的磁片磁碟機，該磁片磁片磁碟機包含的容量層級的客戶內容。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-112">Microsoft servers use BitLocker to encrypt the disk drives containing customer content at the volume-level.</span></span> <span data-ttu-id="6fe2d-113">BitLocker 所提供的加密，可在其他進程或控制項中的失誤時保護客戶內容 (例如，對包含客戶內容) 之磁片的存取控制或回收可能會造成未授權實體存取的情形。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-113">The encryption provided by BitLocker protects customer content in case of lapses in other processes or controls (for example, access control or recycling of hardware) that could lead to unauthorized physical access to disks containing customer content.</span></span>

<span data-ttu-id="6fe2d-114">Exchange Online、Microsoft 小組、SharePoint 線上和商務 OneDrive 也會在應用層使用服務加密，以加密客戶內容。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-114">Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business also use Service Encryption at the application layer to encrypt customer content.</span></span> <span data-ttu-id="6fe2d-115">服務加密在增強式加密保護之上提供版權保護和管理功能。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-115">Service Encryption provides rights protection and management features on top of strong encryption protection.</span></span> <span data-ttu-id="6fe2d-116">這種方式也可讓您在 Windows 作業系統與這些作業系統儲存或處理的客戶資料之間進行分隔。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-116">It also allows for separation between Windows operating systems and the customer data stored or processed by those operating systems.</span></span>

## <a name="how-does-microsoft-365-encrypt-data-in-transit"></a><span data-ttu-id="6fe2d-117">Microsoft 365 如何加密資料傳輸？</span><span class="sxs-lookup"><span data-stu-id="6fe2d-117">How does Microsoft 365 encrypt data-in-transit?</span></span>

<span data-ttu-id="6fe2d-118">Microsoft 365 產品和服務使用強大的傳輸通訊協定（例如 TLS），防止未授權的協力廠商在客戶資料移動到網路時遭到竊聽。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-118">Microsoft 365 products and services use strong transport protocols, such as TLS, to prevent unauthorized parties from eavesdropping on customer data while it moves over a network.</span></span> <span data-ttu-id="6fe2d-119">中轉中的資料範例包括正在傳遞的郵件訊息、線上會議中進行的交談，或是資料中心之間複寫的檔案。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-119">Examples of data in transit include mail messages that are in the process of being delivered, conversations taking place in an online meeting, or files being replicated between datacenters.</span></span>

<span data-ttu-id="6fe2d-120">在 Microsoft 365 中，每當使用者的裝置與 Microsoft 伺服器通訊或 Microsoft 伺服器與另一部伺服器通訊時，資料就會被視為「傳輸」。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-120">In Microsoft 365, data is considered 'in transit' whenever a user's device is communicating with a Microsoft server or a Microsoft server is communicating with another server.</span></span> <span data-ttu-id="6fe2d-121">Exchange Online、SharePoint 線上、Microsoft 團隊和 Office Online 都使用 TLS，以確保資料在傳輸期間保持機密。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-121">Exchange Online, SharePoint Online, Microsoft Teams, and Office Online all use TLS to ensure data remains confidential while in transit.</span></span>

## <a name="how-does-microsoft-365-manage-the-keys-used-for-encryption"></a><span data-ttu-id="6fe2d-122">Microsoft 365 如何管理加密所使用的金鑰？</span><span class="sxs-lookup"><span data-stu-id="6fe2d-122">How does Microsoft 365 manage the keys used for encryption?</span></span>

<span data-ttu-id="6fe2d-123">增強式加密的安全性，只是用來加密資料的金鑰。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-123">Strong encryption is only as secure as the keys used to encrypt data.</span></span> <span data-ttu-id="6fe2d-124">Microsoft 會使用自己的安全性憑證來加密資料內傳輸的 TLS 連線。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-124">Microsoft uses its own security certificates to encrypt TLS connections for data-in-transit.</span></span> <span data-ttu-id="6fe2d-125">針對靜態資料，使用完整大量加密金鑰加密 BitLocker 受保護的磁片區，該金鑰是使用大量的主金鑰進行加密，而該金鑰又會系結至伺服器中的受信任平臺模組 (TPM) 。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-125">For data-at-rest, BitLocker-protected volumes are encrypted with a full volume encryption key, which is encrypted with a volume master key, which in turn is bound to the Trusted Platform Module (TPM) in the server.</span></span> <span data-ttu-id="6fe2d-126">BitLocker 會使用 FIPS 相容的演算法，以確保加密金鑰永遠不會以明文儲存或傳送。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-126">BitLocker uses FIPS-compliant algorithms to ensure that encryption keys are never stored or sent over the wire in the clear.</span></span>

<span data-ttu-id="6fe2d-127">服務加密為客戶在 Exchange Online、Microsoft 小組、SharePoint 線上和商務 OneDrive 中的客戶資料提供額外的加密層級。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-127">Service Encryption provides an additional layer of encryption for customer data-at-rest in Exchange Online, Microsoft Teams, SharePoint Online, and OneDrive for Business.</span></span> <span data-ttu-id="6fe2d-128">服務加密為客戶提供兩個加密金鑰管理選項： Microsoft 管理金鑰或客戶金鑰。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-128">Service Encryption gives customers two options for encryption key management: Microsoft-managed keys or Customer Key.</span></span> <span data-ttu-id="6fe2d-129">使用 Microsoft 管理的金鑰時，Microsoft 365 服務會自動產生並安全地儲存用於服務加密的根機碼。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-129">When using Microsoft-managed keys, Microsoft 365 services automatically generate and securely store the root keys used for Service Encryption.</span></span>

<span data-ttu-id="6fe2d-130">若客戶需要控制其自己的根加密金鑰，即可利用服務加密與客戶金鑰。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-130">Customers with requirements to control their own root encryption keys can leverage Service Encryption with Customer Key.</span></span> <span data-ttu-id="6fe2d-131">使用客戶金鑰，客戶可以使用內部部署硬體服務模組 (HSM) 或 Azure Key Vault (AKV) 來產生自己的加密金鑰。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-131">Using Customer Key, customers can generate their own cryptographic keys using either an on-premises Hardware Service Module (HSM) or Azure Key Vault (AKV).</span></span> <span data-ttu-id="6fe2d-132">客戶根機碼儲存在 AKV 中，而這些機碼是用來加密客戶信箱資料或檔案的其中一個 keychains 的根。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-132">Customer root keys are stored in AKV, where they can be used as the root of one of the keychains that encrypts customer mailbox data or files.</span></span> <span data-ttu-id="6fe2d-133">客戶根金鑰只可透過 Microsoft 365 服務程式代碼以間接方式存取，以進行資料加密，而且無法由 Microsoft 員工直接存取。</span><span class="sxs-lookup"><span data-stu-id="6fe2d-133">Customer root keys can only be accessed indirectly by Microsoft 365 service code for data encryption and cannot be accessed directly by Microsoft employees.</span></span>
