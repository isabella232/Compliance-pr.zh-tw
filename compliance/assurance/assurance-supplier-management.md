<span data-ttu-id="7c454-110">SSPA 是 Microsoft 採購、公司外部和法律事務之間的合作關係，以及公司的安全性，以確保供應商遵循 Microsoft 的隱私權和安全性原則。</span><span class="sxs-lookup"><span data-stu-id="7c454-110">SSPA is a partnership between Microsoft Procurement, Corporate External and Legal Affairs, and Corporate Security to ensure suppliers adhere to Microsoft's privacy and security principles.</span></span> <span data-ttu-id="7c454-111">SSPA 的範圍涵蓋所有處理個人資料或 Microsoft 機密資料的供應商。</span><span class="sxs-lookup"><span data-stu-id="7c454-111">The scope of SSPA covers all suppliers that process Personal Data or Microsoft Confidential Data.</span></span> <span data-ttu-id="7c454-112">SSPA 計畫登記包含符合 Microsoft 的資料保護需求 (DPR) 。</span><span class="sxs-lookup"><span data-stu-id="7c454-112">The SSPA program enrollment includes adherence to Microsoft's Data Protection Requirements (DPR).</span></span> <span data-ttu-id="7c454-113">DPR 包含的安全性和隱私權控制提供者必須先執行，然後才開始與 Microsoft 開始簽約使用。</span><span class="sxs-lookup"><span data-stu-id="7c454-113">The DPR consist of security and privacy controls that suppliers must implement before beginning contracted work with Microsoft.</span></span> <span data-ttu-id="7c454-114">所有已註冊的供應商會自我證明，以每年符合 DPR。</span><span class="sxs-lookup"><span data-stu-id="7c454-114">All enrolled suppliers self-attest to compliance with the DPR annually.</span></span>

<span data-ttu-id="7c454-115">DPR 需求的適用範圍是以六種不同的資料處理類別為基礎，可在 SSPA 的登記中核准。</span><span class="sxs-lookup"><span data-stu-id="7c454-115">DPR requirements are scoped based on six distinct data processing categories a supplier can be approved for as part of their enrollment in SSPA.</span></span> <span data-ttu-id="7c454-116">這些類別是用來識別與提供者提供給 Microsoft 的服務相關的風險。</span><span class="sxs-lookup"><span data-stu-id="7c454-116">These categories are used to identify the risk associated with the services a supplier provides to Microsoft.</span></span> <span data-ttu-id="7c454-117">供應商的資料處理設定檔會決定哪些 DPR 控制項會被視為範圍內，以提供適當的資料保護。</span><span class="sxs-lookup"><span data-stu-id="7c454-117">The supplier's data processing profile determines which DPR controls are considered in-scope to provide appropriate data protection.</span></span> <span data-ttu-id="7c454-118">處理視為較高風險之資料的供應商必須符合所有的 DPR 需求，而且可能也需要提供獨立的相容性驗證。</span><span class="sxs-lookup"><span data-stu-id="7c454-118">Suppliers who process data that is considered a higher risk must comply with all DPR requirements and may also need to provide independent verification of compliance.</span></span> <span data-ttu-id="7c454-119">Microsoft 購買工具會驗證所有供應商的 SSPA 狀態，包括與 DPR 適用的部分相容性，在允許該供應商的採購之前。</span><span class="sxs-lookup"><span data-stu-id="7c454-119">Microsoft purchasing tools validate the SSPA status of all suppliers, including compliance with applicable portions of the DPR, prior to allowing the procurement of that supplier.</span></span>

## <a name="what-types-of-subprocessors-provide-services-for-microsoft"></a><span data-ttu-id="7c454-120">哪些類型的 subprocessors 可為 Microsoft 提供服務？</span><span class="sxs-lookup"><span data-stu-id="7c454-120">What types of subprocessors provide services for Microsoft?</span></span>

<span data-ttu-id="7c454-121">「Subprocessor ' 是 Microsoft 雇傭的協力廠商，其職責包含處理 Microsoft 為處理器的 Microsoft 個人資料。</span><span class="sxs-lookup"><span data-stu-id="7c454-121">A 'subprocessor' is a third party that Microsoft engages whose duties include processing Microsoft Personal Data for which Microsoft is a processor.</span></span> <span data-ttu-id="7c454-122">Microsoft 的 subprocessors 分為三個不同的類別。</span><span class="sxs-lookup"><span data-stu-id="7c454-122">Microsoft's subprocessors fall into three distinct categories.</span></span> <span data-ttu-id="7c454-123">每個客戶都必須先示範 SSPA，才可代表在 Microsoft 上處理客戶資料。</span><span class="sxs-lookup"><span data-stu-id="7c454-123">Each must demonstrate compliance with the SSPA before they can process customer data on Microsoft's behalf.</span></span>

- <span data-ttu-id="7c454-124">**技術** subprocessors 提供用來傳遞特定 Microsoft online 服務的技術。</span><span class="sxs-lookup"><span data-stu-id="7c454-124">**Technology** subprocessors provide technologies used to deliver specific Microsoft online services.</span></span> <span data-ttu-id="7c454-125">如果客戶部署其中一個服務，則針對該服務所識別的 subprocessors 可能會在協助提供該服務時處理、儲存或存取客戶資料或個人資料。</span><span class="sxs-lookup"><span data-stu-id="7c454-125">If a customer deploys one of these services, the subprocessors identified for that service may process, store, or otherwise access customer data or personal data while helping to provide that service.</span></span>
- <span data-ttu-id="7c454-126">**輔助** subprocessors 提供服務，以支援、運作及維護線上服務。</span><span class="sxs-lookup"><span data-stu-id="7c454-126">**Ancillary** subprocessors provide services that support, operate, and maintain the online services.</span></span> <span data-ttu-id="7c454-127">如果客戶部署其中一項服務，識別的 subprocessors 可能會處理、儲存或存取有限的客戶資料或個人資料，同時提供其輔助服務。</span><span class="sxs-lookup"><span data-stu-id="7c454-127">If a customer deploys one of these services, the subprocessors identified may process, store, or otherwise access limited customer data or personal data while providing their ancillary services.</span></span>
- <span data-ttu-id="7c454-128">**員工充實** subprocessors 採用兩種不同的形式：在這兩種情況下，個人資料只會在 microsoft 系統上，且受限於 microsoft 的原則和監督工作。</span><span class="sxs-lookup"><span data-stu-id="7c454-128">**Staff Augmentation** subprocessors take two different forms: In both scenarios, Personal Data resides only in Microsoft facilities, on Microsoft systems, and is subject to Microsoft policies and supervision.</span></span>

    - <span data-ttu-id="7c454-129">第一種形式的人員充實可讓員工支援、運作及維護 Microsoft 線上服務。</span><span class="sxs-lookup"><span data-stu-id="7c454-129">The first form of staff augmentation provides staff that supports, operates, and maintains Microsoft online services.</span></span> <span data-ttu-id="7c454-130">在履行其職責時，這些 subprocessors 可能會對客戶資料或個人資料公開。</span><span class="sxs-lookup"><span data-stu-id="7c454-130">While fulfilling their responsibilities, these subprocessors may be exposed to customer data or personal data.</span></span> <span data-ttu-id="7c454-131">例如，subprocessor 可能會在 Microsoft server 上執行遠端疑難排解，但這樣做可能會在伺服器損毀傾印記錄檔中對客戶資料的程式碼片段公開。</span><span class="sxs-lookup"><span data-stu-id="7c454-131">For example, a subprocessor may perform remote troubleshooting on a Microsoft server and while doing so may be exposed to snippets of customer data in a server crash dump log.</span></span>
    - <span data-ttu-id="7c454-132">第二種形式的人員擴充包括 subprocessors 哪些人員與 Microsoft 全職員工共同作業，以支援、操作和維護 Microsoft 線上服務。</span><span class="sxs-lookup"><span data-stu-id="7c454-132">The second form of staff augmentation involves subprocessors who work side by side with Microsoft full-time employees to support, operate, and maintain Microsoft online services.</span></span> <span data-ttu-id="7c454-133">在 Microsoft 全職員工以外的工作中，您可能會將這些 subprocessors 公開給假名化資料。</span><span class="sxs-lookup"><span data-stu-id="7c454-133">These subprocessors may be exposed to pseudonymized data as part of their work alongside Microsoft full-time employees.</span></span>

<span data-ttu-id="7c454-134">若要遵循 Microsoft 的資料保護需求 (DPR) ，必須使用技術和輔助 subprocessors 來執行存取控制。</span><span class="sxs-lookup"><span data-stu-id="7c454-134">Technology and Ancillary subprocessors are required to implement access controls in compliance with Microsoft's Data Protection Requirements (DPR).</span></span> <span data-ttu-id="7c454-135">這些需求符合或超過 Microsoft 對其客戶在線上服務條款 (OST) 中所做的合約承諾。</span><span class="sxs-lookup"><span data-stu-id="7c454-135">These requirements meet or exceed the contractual commitments Microsoft makes to its customers in the Online Service Terms (OST).</span></span> <span data-ttu-id="7c454-136">從事人員充實工作的供應商，必須對 Microsoft 全職員工使用相同的存取控制措施。</span><span class="sxs-lookup"><span data-stu-id="7c454-136">Suppliers who perform staff augmentation work are subject to the same access controls in place for Microsoft full-time employees.</span></span>
